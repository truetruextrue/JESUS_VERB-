## VISÃO GERAL

A pasta `ARQUETIPOS/` é o coração pulsante do KOBLLUX TRINITY SYSTEM. Ela contém
**12 módulos web autônomos**, um para cada arquétipo CADIAL. Cada módulo é um
**organismo completo** — com seu próprio `index.html`, folhas de estilo (`css/`),
scripts (`js/`), assets visuais e frequência ressonante própria.

Quando o usuário seleciona um arquétipo na matriz ZPR do `index.html` principal,
o **BLLUE cadial-coupler.js** (0×02 · INTEGRAR) carrega o módulo correspondente
no iframe central. O carregamento é isolado — cada arquétipo vive em seu próprio
contexto, sem interferência dos demais.

A pasta `_state/` persiste o estado global do sistema: qual arquétipo está ativo,
histórico de navegação, configurações de frequência e dados de sessão.

## ESTRUTURA DE CADA MÓDULO

Cada uma das 12 pastas de arquétipo segue a mesma estrutura base:

```
{arquetipo}/
├── index.html          → Shell principal do módulo (autônomo, carregável em iframe)
├── css/
│   ├── main.css        → Estilos base do arquétipo (paleta de cor, tipografia)
│   └── animations.css  → Animações ressonantes na frequência do arquétipo
├── js/
│   ├── core.js         → Lógica central do arquétipo
│   ├── voice.js        → Configuração de voz TTS específica do arquétipo
│   └── data.js         → Dados, frases e perfil do arquétipo
├── assets/
│   ├── symbol.svg      → Símbolo geométrico do arquétipo em SVG
│   └── orb.svg         → Avatar ORB determinístico do arquétipo
└── manifest.json       → Declaração de identidade do módulo
```

**Exemplo — `atlas/manifest.json`:**

```json
{
  "name": "ATLAS",
  "opcode": "0x0A",
  "symbol": "⬡",
  "hz": 528,
  "action": "ESTRUTURAR",
  "color_primary": "#1a4a6e",
  "color_accent":  "#00d4ff",
  "essence": "O estrategista que sustenta mundos inteiros",
  "voice_pitch": 0.85,
  "voice_rate": 0.92,
  "cadial_version": "7.9"
}
```

## ESTRUTURA `_state/`

A pasta `_state/` é o **registro de memória viva** do sistema CADIAL. Ela persiste
o estado entre sessões, garantindo continuidade da experiência.

```
_state/
├── active_arch.json     → Arquétipo ativo no momento (nome, hz, opcode)
├── session_history.json → Histórico de arquétipos visitados na sessão
├── user_prefs.json      → Preferências do usuário (voz, tema, frequência base)
├── resonance_map.json   → Mapa de ressonância entre arquétipos visitados
└── pulse_log.json       → Log de pulsações e eventos do sistema
```

**Exemplo — `_state/active_arch.json`:**

```json
{
  "arch"     : "atlas",
  "symbol"   : "⬡",
  "hz"       : 528,
  "opcode"   : "0x0A",
  "activated": "2026-05-14T00:00:00Z",
  "session"  : "kobllux-delta7"
}
```

## ADICIONANDO UM NOVO MÓDULO

Para integrar um novo arquétipo ao sistema CADIAL:

1. **Criar a pasta** `ARQUETIPOS/{nome}/` com a estrutura base
2. **Declarar** o arquétipo em `kobllux/archetypes.py`
3. **Criar o perfil** em `kobllux/profiles/{nome}.json`
4. **Registrar** na lista `BLLUE.archetypes` em `web/js/modules/cadial-coupler.js`
5. **Adicionar** ao `web/js/modules/screensData.js` (dados da matriz ZPR)
6. **Executar** `cadial_scan.py` para re-indexar (0×01 · DETECTAR)
